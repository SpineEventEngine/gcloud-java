Namespace

@Immutable
class Namespace

A value object representing the Datastore namespace.

The primary usage of the namespaces is multitenancy.

By default, a namespace constructed from a TenantId has a single capital letter prefix reflecting the type of TenantId. These prefixes are:

  • D - for "Internet Domain";
  • E - for "Email";
  • V - for "String Value".

The framework users may override the behavior by registering a custom NamespaceConverter. If registered, no defaults are applied.

One should register a NamespaceConverterif and only if the target Datastore instance already contains namespaces to work with.

Basically, one of three scenarios in working with the Datastore namespaces takes place.

  • There are no namespaces in the Datastore at all. All the namespace manipulations are performed by the means of the framework using the defaults.
  • All the present namespaces start with one of the prefixes listed above. In this case the described default TenantId conversion behavior is applied.
  • A custom NamespaceConverter is registered, and its implementation is consistent with the namespaces (if any) already present in the Datastore instance.
