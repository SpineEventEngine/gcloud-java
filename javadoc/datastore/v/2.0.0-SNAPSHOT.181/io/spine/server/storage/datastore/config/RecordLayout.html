Package io.spine.server.storage.datastore.config

Class RecordLayout

  • All Implemented Interfaces:
    @SPI() 
public abstract class RecordLayout<I, R extends Message>

    Describes the layout of Datastore Entities in which the Message records of certain type are stored.

    Datastore allows to store Entities in flat structure or by grouping the Entities into Entity groups according to ancestor-child relations.

    Descendants may define their own layout for records and then plug it into their storage factory via DatastoreStorageFactory.newBuilder().organizeRecords(typeOfRecord, recordLayout).

      • ancestorFilter

         abstract Optional<StructuredQuery.Filter> ancestorFilter(RecordQuery<I, R> query, DatastoreMedium datastore)

        If applicable, creates a Datastore ancestor filter by analyzing the passed record query.

        Such a special filtering is required by Datastore, so that it could efficiently fetch the records as children of some ancestor. Descendants of RecordLayout must supply such a Datastore-specific filtering criterion in case their storage layout is hierarchical.

        If the structure of storage is flat (i.e. no ancestor-child relations are established), this method returns Optional.empty().

        Parameters:
        query - record query to create an ancestor filter from
        datastore - Datastore connector
        Returns:

        a new Datastore ancestor filter, or Optional.empty() if such an organizational is not applicable for the current type of RecordLayout
